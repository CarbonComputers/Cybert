# Contributing to Cybert

Thank you for your interest in contributing to the Cybert project!  
We welcome community involvement ‚Äî whether you're fixing bugs, improving documentation, refining 3D models, or expanding compatibility.

Before contributing, please read the guidelines below.

---

## üß≠ How to Contribute

### 1. Fork the Repository
Click the **Fork** button at the top-right of the GitHub page to create your own copy of the project.

### 2. Create a Feature Branch
Use descriptive names such as:
feature/add-new-display-driver
fix/keyboard-mapping-issue
docs/improve-assembly-guide

### 3. Make Your Changes
Follow the project structure and coding style as closely as possible.  
If modifying hardware files, clearly document:
- What changed  
- Why  
- Impact on compatibility  

### 4. Test Your Contributions
Please test your changes on actual hardware whenever applicable:
- PCB changes should include layout notes  
- Display configuration changes should include screenshots  
- Firmware changes should include compile logs  

### 5. Submit a Pull Request (PR)
Your PR should include:
- A clear title  
- A brief summary of changes  
- Screenshots or diagrams if needed  
- Any known issues or limitations  

The maintainers will review your pull request and may ask for adjustments.
